## HIGH

### H1 — The credentialed-release writer is unwired (go-live functional blocker)
**Severity:** High · **Location:** `apps/seo/src/lib/review/signoff.ts:80,117` (`recordClientSignoff`/`recordCredentialedRelease`), `apps/seo/src/lib/activation.ts:129` (`isPilot`)
**Finding (architecture H1 + convention A.006.1, agreeing):** `recordCredentialedRelease` is the ONLY writer of `credentialed_releases` — the sole record `canPublish()` accepts — and it has **zero non-test callers**; `isPilot()` likewise has zero consumers. The publish route READS releases but nothing WRITES one. Consequence: with full activation (creds + `PUBLISH_ENABLED` + a real reviewer), a piece still cannot reach `published` because no route records the release the FSM requires. Fail-closed (not a security hole) and the DR-037 `pilot:false` guard holds trivially (never executes) — but it is a go-live-blocking functional gap. This is audit-005 **M1**/**A.005.1's wiring half** re-confirmed at runtime: PR #63 delivered the predicate widening, not the wiring.
**Recommendation → A.006.1 (build, HOLD for human merge — production-critical + DR-037):** wire `recordCredentialedRelease`/`recordClientSignoff` into the review→release route, passing `pilot: isPilot()`; test (a) release persisted on approval, (b) `placeholder-in-production` refusal when `VERCEL_ENV==='production'`. Until then `go-live-checklist.md` must state: go-live cannot publish even after the env flip.

## MEDIUM

### M1 — SoM live Gateway call site outside `gate-path-lint` scope (metering coverage gap)
**Location:** `packages/core/src/ai/som-direct-runner.ts:108`; lint at `packages/core/src/gates/gate-path-lint.ts:106,129`
**Finding (architecture M1):** `som-direct-runner` correctly uses `resolveGatewayModel(..., { forceGateway: true })` (metered today), but `gate-path-lint` (the audit-005 H4 fix) globs only `packages/core/src/gates/*-gate.ts`. A future edit dropping `forceGateway` here would silently reopen the direct-Anthropic BYOK branch and escape the D4 cost ledger — exactly the invariant H4 protects, now applicable to a second metered call site.
**Recommendation → A.006.2 (build):** extend the lint's discovery to scan model-resolving files under `packages/core/src/ai/` (or a metered-call-site list) with the same `forceGateway`-required assertion.

### M2 — New cron routes omit the repo-standard route config
**Location:** `apps/seo/src/app/api/cron/ingest-share-of-model/route.ts`, `.../freshness-scan/route.ts`
**Finding (convention A.006.2):** every other route declares `export const runtime = "nodejs"` (15/15); request-dynamic routes also declare `dynamic`/`maxDuration`. Both new cron routes declare none, despite `server-only` + `process.env` + (live) metered model calls + DB writes — a convention break on scheduled entrypoints and a latent runtime/timeout risk.
**Recommendation → A.006.3 (build):** add `runtime="nodejs"`, `dynamic="force-dynamic"`, explicit `maxDuration` (ingest probes N engines → generous) to both cron routes.

### M3 — `defaultPublishEnabled` diverges from the activation predicate
**Location:** `apps/seo/src/app/content/api/publish/route.ts:71-73`
**Finding (convention A.006.3 + architecture L1):** `DEFAULT_DEPS.publishEnabled` reads only `CONTENT_PUBLISH_ENABLED==="1"`; the canonical `activationPublishEnabled()` honors both `PUBLISH_ENABLED`/`CONTENT_PUBLISH_ENABLED` AND requires creds. The live POST overrides with the canonical one (so no live risk — only `DEFAULT_DEPS` callers/tests hit the weak default), but two predicates for "is publishing on" is a footgun at go-live.
**Recommendation → fold into A.006.1 (same file):** make `DEFAULT_DEPS.publishEnabled` delegate to `activationPublishEnabled` — one source of truth.

### M4 — SoM `source_channel` hybrid split not reflected in the spec/migration comments
**Location (code, shipped):** `apps/seo/src/lib/metrics/som-adapters/types.ts:62` (`direct-citation|direct-proxy|vendor`), `som-adapters/base.ts`, `som-direct-runner.ts`, `som-live-store.ts`; governing: DR-038 addendum. **Location (spec, stale):** `engineering-rfc.md:138,723` (`source_channel (direct·vendor-api)`), PRD §3.4/§16 (silent on proxy-vs-citation); `packages/schema-flywheel/drizzle/0039_seo_cost_ledger.sql:27-28,131-132,149` (comments say "Gateway direct-query").

Documentation-only (no schema/runtime bug).
**Recommendation → A.006.4 (build, docs):** update RFC §3.1 table + dashboard text + PRD §3.4/§16 to the 3-channel set + the proxy-labeling rule; refresh `0039` + matching `packages/schema-flywheel/src/content.ts` comments. Do NOT alter the DDL/default.

## CALIBRATION (→ manifest structured check)

### `live-side-effect-idempotency` (HIGH-default; state-historian §4, #72 re-judge)
`share_of_model` has no UNIQUE/ON CONFLICT (correct for an append-only time series), but the manual DoD-close ingest wrote 30 rows with no row-level idempotency key — a second accidental run would silently double the WW client's citation counts; correctness rests entirely on running the slice exactly once. The judge reviewed the (inert) cron, never the manual ingest. **→ add structured check `live-side-effect-idempotency`:** any run performing an external write to a live store outside a merged-and-tested code path (manual ingest/backfill/seed) must STATE the idempotency key/dedup mechanism, or the run-exactly-once operator guard + why a DB constraint is absent.
