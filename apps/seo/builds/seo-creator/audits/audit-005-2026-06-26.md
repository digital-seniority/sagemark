## HIGH findings (→ active risks; correctives queued A.005.x / manifest)

### H1 — DR-025's binding "PR 020 MUST" obligations unmet by #58 (spec-reconciler 1.2)
`DR-025` lists three things PR 020 MUST do: (1) add the `gate_results` table + wire `getGateResult`; (2) populate `eval_ran` on every gate run; (3) **widen `PersistedAuthorization` (`context.ts:136-140`) to include `granted_at` + `scope`.** #58 did none of (1)/(3): no table (see H2), and `PersistedAuthorization` still carries only `{id, revokedAt, expiresAt}` — so the §11.5 YMYL release predicate cannot evaluate `scope` and treats "granted" implicitly. DR-025 was never discharged.
**Disposition:** ratified in **[[DR-039]]** (gate_results stays a seam projection in v1; supersedes DR-025's table mandate) + **the `PersistedAuthorization` `granted_at`/`scope` widening is the load-bearing remainder → corrective A.005.1.**

### H2 — `gate_results` shipped as a seam projection, not a persisted table (spec-reconciler 1.1)
RFC §3.1/§3.3 + PRD §6/§9.1 describe `gate_results` as a first-class queryable audit table (with `idx(sourcing_blocked)` driving the D3 metric). #58 computes the sourcing-block rate from a seam projection of existing gate-result data and persists no audit row (`0039` header explicitly: "there is NO `gate_results` table … it is a seam-level projection").
**Disposition:** **[[DR-039]]** records the seam-projection-in-v1 decision; **spec-update A.005.3** to reconcile RFC §3.1/§3.3 + PRD §6/§9.1.

### H3 — `token-scope.test.ts:279-284` hollow Tier-2 body, now un-gated by the live `DATABASE_URL` (test-quality)
The `[T2]` body is `assert.ok(true)` — no live check — gated only by `HAS_DB = Boolean(process.env.DATABASE_URL)`. **`DATABASE_URL` is now set**, so locally this test **runs and passes vacuously** while claiming to prove "anon reaches ZERO rows on review_tokens + comment_threads" (the load-bearing anon-isolation proof for P1.C.1's tokenized review tables). In CI it skips (turbo doesn't pass `DATABASE_URL`), so it asserts nothing anywhere.
**Disposition:** corrective **A.005.2** — replace the `assert.ok(true)` body with a real live anon-zero-rows check (mirror `rls-contract.test.ts`'s psql/pg runner) OR a fail-loud `throw NEEDS-INPUT`; + add a dedicated CI `node --test token-scope.test.ts` step.

### H4 — gate-path-lint scans a HARDCODED 2-file list (no auto-discovery) (state-historian, calibration)
`gate-path-lint.ts:93-96` `GATE_PATH_SOURCE_FILES` is a fixed 2-file list. A future `packages/core/src/gates/*-gate.ts` calling `resolveGatewayModel(GATE_MODEL, "host")` WITHOUT `forceGateway` is never scanned → its model calls fall through to the direct-Anthropic BYOK branch, **escaping the cost ledger** (the DR-013 invariant #54 protects + the billing basis for #58). Runtime enforcement is sound; the lint's *coverage* is open.
**Disposition:** **manifest structured-check added this audit** (`gate-metering-lint-coverage-complete`, below) + code corrective **A.005.2** to make `gate-path-lint` glob-discover all gate files (and resist aliased/wrapped imports).
