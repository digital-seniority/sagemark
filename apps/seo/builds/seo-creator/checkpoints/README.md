# SEO Creator Build Checkpoints

This directory holds per-run checkpoint files written by `/seo-creator-build` at the end of each invocation.

## Filename convention

```
run-NNN-checkpoint.md
```

Where `NNN` is the zero-padded run number (001, 002, ...). One checkpoint per run.

## Content contract

Each checkpoint is a Markdown file generated by Phase 7 (self-eval retro) of the orchestrator. It captures:

- Run number + timestamp
- PRs touched (id, title, lane, end state — MERGED / PR_CREATED / BLOCKED / PREVIEW_FAILED / INTERRUPTED)
- Judge findings (per PR)
- Drift signal (per anchor, if drift-control is on)
- Resource utilization (if floodgate-resource-aware)
- Recent learnings — appended to STATE.md until they roll off the last-5 window
- One-line summary for run-log.md

The exact template lives at `C:/Users/stone/Code/sagemark/.claude/skills/seo-creator-build/checkpoint-template.md` — copy that on each run.

## Why one file per run

Each checkpoint is small (1-3 pages); the full history is browsable by listing the directory. STATE.md is the latest-only view; run-log.md is the one-line-per-run audit trail; checkpoints/ is the full-detail per-run record.

## Append-only

Once a checkpoint is written, do not modify it. If a correction is needed, add a follow-up checkpoint (`run-NNN-checkpoint-correction.md`) that supersedes the original. The audit trail is the value.
